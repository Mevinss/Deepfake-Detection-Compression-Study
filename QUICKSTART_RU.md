# 🚀 Быстрый старт — Детекция Дипфейков (2026)

Это руководство поможет вам быстро запустить веб-приложение для детекции дипфейков с современным дизайном и поддержкой GIF.

---

## ⚡ Установка и запуск за 5 минут

### Шаг 1: Клонирование и установка

```bash
# Клонируем репозиторий
git clone https://github.com/Mevinss/Deepfake-Detection-Compression-Study.git
cd Deepfake-Detection-Compression-Study

# Создаем виртуальное окружение
python -m venv .venv

# Активируем (Linux/Mac)
source .venv/bin/activate
# Или на Windows:
# .venv\Scripts\activate

# Устанавливаем зависимости
pip install -r requirements.txt
```

### Шаг 2: Инициализация модели

```bash
# Создаем рабочую модель с предобученными весами ImageNet
python download_model.py --model mobilenetv3
```

Это создаст базовую модель для демонстрации. Для лучшей точности обучите модель на датасете дипфейков (см. ниже).

### Шаг 3: Запуск веб-приложения

```bash
# Запускаем сервер
python app.py
```

**Готово!** Откройте браузер и перейдите на: **http://127.0.0.1:5000**

## 📊 Обучение модели на датасете

Приложение работает с предобученными весами ImageNet, но для лучшей точности обязательно обучите модель:

### 1. Подготовка данных

```bash
# Автоматическая подготовка структуры
python download_dataset.py

# Следуйте инструкциям в data/README.md для загрузки датасета
```

Рекомендуемые датасеты:
- **Celeb-DF v2**: https://github.com/yuezunli/celeb-deepfakeforensics
- **FaceForensics++**: https://github.com/ondyari/FaceForensics
- **DFDC**: https://www.kaggle.com/c/deepfake-detection-challenge/data

### 2. Обучение модели

```bash
# После добавления данных в data/train/, data/val/, data/test/
python src/train.py --config configs/config.yaml

# Или без PyTorch Lightning:
python src/train.py --config configs/config.yaml --no_lightning
```

### 3. Перезапуск приложения

```bash
# Веб-приложение автоматически загрузит обученную модель
python app.py
```

## 🔧 Настройка

### Изменение порта

Отредактируйте последнюю строку в `app.py`:

```python
app.run(host='0.0.0.0', port=8080, debug=False)  # Измените 5000 на 8080
```

### Выбор модели

Отредактируйте `configs/config.yaml`:

```yaml
model:
  name: mobilenetv3  # mobilenetv3 (быстрая) | efficientnet_b0 (точная) | ghostnet (баланс)
```

### Настройка GPU/CPU

Приложение автоматически определяет доступность CUDA.  
Для принудительного использования CPU:

```python
# В app.py измените:
device = torch.device("cpu")  # Вместо auto-detection
```

## ❓ Частые вопросы

### Нужен ли датасет для запуска?

**Нет!** Веб-приложение работает с предобученными весами без датасета.  
Датасет нужен только для обучения своей модели для лучшей точности.

### Какие видео форматы поддерживаются?

**Видео:** MP4, AVI, MOV, MKV, WEBM  
**Изображения:** GIF (анимированные)  
**Максимальный размер:** 100 МБ

### Как долго анализируется видео?

- CPU: 30-60 секунд
- GPU (CUDA): 10-20 секунд

### Можно ли использовать без GPU?

**Да!** Приложение работает на CPU, но медленнее.

### Где хранятся загруженные видео?

Видео временно сохраняются в `uploads/` и автоматически удаляются после обработки.

## 🐛 Решение проблем

### Ошибка: "No module named 'src'"

```bash
export PYTHONPATH="${PYTHONPATH}:$(pwd)"
```

### Ошибка: "Face detector not initialized"

```bash
pip install mediapipe
```

### Ошибка: "CUDA out of memory"

Уменьшите `max_frames` в `app.py`:

```python
max_frames=15,  # Вместо 30
```

### Приложение не открывается

Проверьте, что порт 5000 не занят:

```bash
# Linux/Mac:
lsof -i :5000

# Windows:
netstat -ano | findstr :5000
```
